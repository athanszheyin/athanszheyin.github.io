30分钟建立静态博客指南

2019-06-03

作为 Hello World 开篇博文，简单记录一下建站的步骤和所需技术栈👇

  • GitHub Page作为博客后端服务器
  • 基于Node.js的Hexo作为前端框架
  • 使用Gridea作为用户界面来更新维护博客

一个完美的MVC架构，而且简易实用，便于上手。

Modeler：GitHub Pages

作为全球最大同性交友平台，Github于三年前推出免费的个人页面 GitHub Pages 方便各位技术宅展示自己。三年来Github pages仍然是我所知道的唯一便捷的blog搭建平台，同时空间免费，流量免费。Github被微软收购之后，我推测在可预见的未来这些福利将会持续下去。因此如果你要我推荐一个博客平台，我将自信地安利给你GitHub Pages。具体建立自己的Github Pages的方法一目了然，参阅官方教程即可。如果你已经有GitHub账户并且熟悉Git commands，整个过程不会超过 5分钟 。

Controler：Hexo

完成GitHub Pages的设置后，会收到Jekyll模板植入的提示。即你编辑的html/css文件会经过Jekyll的静态编辑生成网页。但是我并不推荐这一套流程，因为Jekyll模板太过繁琐，创建页面编辑页面不太直观。基于Hexo的静态网页生成器就要灵活得多，主题数量也很丰富。安装 Hexo 需要预装 Node.js ,整个过程也是无比丝滑，预计费时 10分钟 。

我发现hexo同时也是美国一家上市医药公司的ticker，该公司应属于小盘股，最近走势不错，可是医药股一向波动较大，投资需谨慎。

Viewer: Gridea

Gridea 是我最近找到的一个开源博客编辑交互界面，与hexo无缝链接，甚至可以省略刚才的 10分钟 Hexo配置时间。基本上下载安装，然后在配置界面和GitHub Pages和repository对接起来即可。但是使用Gridea的缺憾在于放弃了Github的双向同步特性，Gridea会在本地保存好你编辑的网页文件，然后单向推送给Github，如果你通过其他途径修改了Github repo上的文件，本地并不会收到这些更新。所以如果你可能在多个终端上更新和编辑博客，那么你需要使用一个双向同步的工具（比如one drive，google drive，dropbox）来保证所有Gridea文件下的一切都是同步的。这最后一步大概耗时 20分钟 。

大多数人会更倾向于自己选择主题建站，甚至申请个性域名。但我本着如无必要，勿增实体的原则，选择了最简洁的法门。博客最重要的是内容，忠实记录自己的生活点滴，成长轨迹。如果你对于Hexo建站感兴趣，以下分享一些有用的链接以飨读者：
